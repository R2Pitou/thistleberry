You are looking at the published site, not the editor. Thistleberry keeps structured source in Git and publishes deterministic HTML to GitHub Pages.

What this page means

If you can see this page, the template deployed correctly and GitHub Pages is working.

This page is only the published output. It does not edit itself. Thistleberry's editor lives in the repository and runs locally.

What to do next

  1. Go back to your repository on GitHub. If Pages is not enabled yet, open Settings then Pages and set Source to GitHub Actions.
  2. Choose how you want to edit. If you want the visual editor, run the local app with npm install and npm run dev, then open http://localhost:3000. If you want the fastest possible change, edit site/index.html directly on GitHub.
  3. Look at the source of this page. The canonical page document lives in .gitscribe/pages/index.json. The published HTML lives in site/index.html.

What Thistleberry is doing

Thistleberry is the translation layer between a visual editor and final static HTML. The repo stays the source of truth. There is no database behind this site.
